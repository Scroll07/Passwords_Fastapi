Security

How Pas CLI handles security

Pas CLI is designed with a local-first security model. The vault never leaves your machine unless you explicitly upload it. The master password never reaches the server. The backend only operates with authenticated requests.

This page documents the specific security properties of the CLI and backend so you understand exactly what is and is not protected.

Master password

Your vault is protected by a master password that you set on first use. This password is used to derive an encryption key locally — it is never transmitted to the backend or stored anywhere on disk. Without the master password, the vault contents cannot be read.

JWT authentication

The backend uses JSON Web Tokens for all protected routes. After logging in with pas api login, you receive a short-lived access token and a longer-lived refresh token. The access token is required for all backup operations. Tokens are stored in your local session config.

Session control

You can invalidate your current session at any time by running pas reset-session. This clears the stored tokens from your local config. To use the API again, you will need to log in. This is useful if you suspect your machine has been accessed or if you are switching environments.

Protected backup actions

All backup endpoints on the backend require a valid JWT access token. Unauthenticated requests are rejected. The backend does not expose any endpoint that reads or writes vault data without authentication.

Vault backup lifecycle

Backups are uploaded and downloaded explicitly by the user using pas api upload and pas api download. There is no automatic sync. You control when your vault is pushed to the API and when it is pulled down. You can also delete backups at any time using pas api delete.

Clipboard safety

When you copy a password using pas copy, the credential is placed in your system clipboard for the duration of the active session. Pas CLI does not automatically clear the clipboard, so treat copied passwords the same way you would treat any sensitive data in the clipboard.

Found a security issue? Open an issue on GitHub or contact the maintainer directly. The project is open source — you can audit the code yourself.
