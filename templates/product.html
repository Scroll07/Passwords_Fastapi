What is Pas CLI

Pas CLI is a command-line password manager designed for developers and technical users who prefer working in the terminal. It stores credentials in a locally encrypted vault, protected by a master password that never leaves your device.

The tool supports an optional FastAPI backend for account-based vault backup sync — useful when you work across multiple machines. The backend is self-hosted and open source.

Who it is for

Pas CLI is built for developers, students, and technical users who are comfortable in the terminal and prefer minimal, scriptable tooling.

  • Developers managing credentials for multiple projects and services.
  • Students learning secure credential handling.
  • Technical users who want a CLI-native workflow without a SaaS dependency.
  • Anyone who wants self-hosted backup control.

Architecture

The project is split into two independent components that can be used together or separately.

CLI Client

A Python command-line tool for local vault operations. Runs entirely on your machine. Handles add, search, copy, edit, delete, export, import, password generation, and session management. Does not require the backend to function.

FastAPI Backend

An optional REST API server for user registration, JWT authentication, and vault backup management. Stores encrypted vault snapshots per user. Enables multi-machine restore via pas api download.

CLI commands

The CLI provides a complete set of local vault operations:

  • Add, edit, and delete password entries.
  • Search entries by name, username, or note.
  • Copy passwords directly to clipboard.
  • Generate strong passwords with configurable length.
  • Export and import vault as JSON.
  • Change master password.
  • Reset session tokens.
  • Manage API connection config.

Backend capabilities

The FastAPI backend handles authentication and backup operations:

  • User registration and login.
  • JWT access and refresh token issuance.
  • Upload and download vault backup snapshots.
  • List and delete existing backups.
  • Protected routes that require valid JWT.
