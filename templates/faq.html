FAQ

Frequently asked questions

Answers to common questions about how Pas CLI works, where data is stored, and what happens in edge cases.

Is the vault stored locally or on a server?

Your vault is stored locally by default. It only reaches the backend if you explicitly upload a backup using the API commands.

Do I need an API account to use Pas CLI?

No. All core local vault features work without an API account. An account is only needed for backup upload, download, and sync-related actions.

What does pas api upload actually do?

It uploads your vault backup to the backend so you can restore it later on another machine or keep an off-device backup copy.

Can I restore my vault on another PC?

Yes. After installing Pas CLI and logging into the API account, you can download your uploaded backup and restore access to your vault.

What is the difference between pas export and pas api upload?

pas export saves a backup locally as a JSON file, while pas api upload sends a backup to the backend for remote storage and later restore.

What happens if I forget my master password?

Without the master password, the local vault cannot be decrypted. That password is not stored or recoverable through the backend.

Is Pas CLI open source?

Yes. The project is open source, so you can inspect the code, audit the implementation, and run the backend yourself.
