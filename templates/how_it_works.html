How It Works

From install to restore in six steps

Pas CLI is designed to be straightforward. You can be up and running with a local vault in under two minutes, and optionally add API backup sync when you need it.

01

Clone and install

Clone the repository and run the setup script. This installs the CLI and sets up your initial configuration. 

git clone https://github.com/Scroll07/password_manager.git
cd password_manager
./realise.sh

02

Register with the API

Optional but recommended for backup sync. Register an account on your self-hosted backend instance. This generates your JWT credentials. 

pas api register

03

Add your first entry

Add a password entry to your local vault. Use --gen to auto-generate a strong password, or -p to set one manually. 

pas add github -u myuser --gen

# or

pas add github -u myuser -p mypassword --note "personal account"

04

Search, copy, and manage entries

Use the CLI to look up credentials, copy passwords to clipboard, edit entries, or delete ones you no longer need. 

pas find gmail.com
pas copy github
pas edit 1 -p newpass
pas del github

05

Export locally or back up to the API

Export your vault to a local JSON file for manual backups, or upload a snapshot to the API for cloud-accessible restore. 

pas export backup.json
pas api upload

06

Restore on another machine

On a new machine, clone the repo, install Pas CLI, log in to the API, and download your vault backup. Your credentials are available immediately. 

pas api login
pas api download

Need the full command reference? The docs page has complete tables for all CLI and API commands, plus backend endpoint documentation.
